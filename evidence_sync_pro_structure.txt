KEY ARCHITECTURAL NOTES:
======================

1. PARSER ARCHITECTURE (Abstract Base Pattern):
   - All parsers inherit from base_parser.BaseParser
   - Input: filepath or raw data
   - Output: List[Event] normalized to common schema
   - Each parser is independent & testable

2. CORRELATION ENGINE (Rule-Based):
   - Rules are pluggable YAML configs
   - Each rule defines input artifact types, matching logic, confidence calculation
   - Rules can be enabled/disabled per investigation
   - Rules output: Correlation{source1, source2, match_type, confidence_score, evidence}

3. CONFIDENCE SCORING (ML + Heuristic Hybrid):
   - ML model (trained on forensic datasets): predicts base reliability (0-100%)
   - Heuristics layer: applies manipulability adjustments
   - Final score = ML_score * artifact_manipulability_factor
   - Explainability: show which factors contributed to final score

4. STORAGE (SQLite + JSON):
   - Case metadata: SQLite (fast queries)
   - Evidence artifacts: JSON (easy export/audit)
   - Chain of custody: Append-only audit log

5. UI/CLI DUAL INTERFACE:
   - GUI for investigators (PyQt6, Windows-native)
   - CLI for automation/batch processing
   - Both share same core engine

6. REPORTING (Multi-Format):
   - PDF: Court-ready narrative + timeline
   - HTML: Interactive timeline + correlation graph
   - JSON: Machine-readable evidence export
   - CSV: Spreadsheet analysis
